Markdown编辑器:Mango

Feb 9, 2017 | 操作记录

前几天，在「Linux(Elementary OS)」中搞完了「Github + Hexo」搭建个人博客，这是链接。这篇文章记录下「Linux」中的「Markdown」编辑器「Mango」

在哪篇文章中发现的「Mango」,我忘记了，在这里对那位作者道谦了，不好意思。

先放张「Mango」的预览图

下载「Mango」

「Mango」,这是「Mango」开发者的项目仓库，在这里可以下载。

安装「Mango」

下载，解压后的内容:

  • 压缩包和解压后的「mango-xxx」文件夹
  • 「mango-xx」文件夹中的内容

其中：
「mango-xxx」文件夹中的「mango」是启动文件，启动方式：

//进入启动文件「mango」所在的文件夹  
./mango

即可。

创建快捷方式

我的系统是「Elementary 0.4」,基于「Ubuntu」的一个「Linux」发行版，其他的「Linux」版本成不成，我不知道，没试过。

  • 创建「xxx.sh」文件,就是「Linux」中的「Bash」文件

    以「mango.sh」为例： 

cd mango-xxx//进入「mang-xxx」文件夹  
vim mango.sh//用「vim」编辑器创建「mango.sh」文件

这是「mango.sh」文件中的内容：

其中：

#bash//对于现在的兼容性来说，写不写都行，只是个说明  
cd ~/mango//进入「mango」文件夹  
&&//连接前后两条命令  
./mango//运行「mango」
  • 创建图标
    网上找个「PNG」格式的图标放在「mango-xxx」文件夹中就好了，名字改简单点

  • 创建快捷方式

    在「/usr/share/applications」文件夹中创建「mango.desktop」文件,名字随意，后缀不变，内容如下： 

[Desktop Entry]  
Encoding=UTF-8  
Version=1.0  
Name=Mango  
GenericName=Mango  
Exec=sh /home/breky/mango/mango.sh  
Terminal=false  
Icon=/home/breky/mango/mango.png  
Type=Application  
Comment=Pycharm IDE  
Categories=Application;

其中：

  1. [Exec=sh /home/breky/mango/mango.sh] 这句，[sh]和后面的[/home/breky/mango/mango.sh]之间有个空格，[breky]是我的用户文件夹，[mango]就是[mango-xxx]文件夹，只是移到这里了。
  2. [Icon=/home/breky/mango/mango.png] 这句是图标

参考文章：Ubuntu16.04下创建桌面快捷方式

以上，完
