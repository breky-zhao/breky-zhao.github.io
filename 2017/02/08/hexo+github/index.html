Hexo + Github

Feb 8, 2017 | 操作记录

搞了这么几天，终于把这个「Hexo」给搞完了，现在把整个操作过程放在这里。

整个过程分这么几个步骤：

  1. 建立「Github」的「repository」
  2. 安装「Hexo」
  3. 安装「Git」
  4. 本地文件上传至「Github」的「repository」中

开始

建立「Github」的「repository」

  1. 创建 Github 账号

  2. 登录后，单击右上角「＋」号，再选择「New Repository」

  3. 「New Repository」详细内容

    其中:

    「Rpository Name」: yourusrname.github.io，比如: breky-zhao.github.io

    勾选「initialize this repository with a README」，其实吧，选不选无所谓，后期如果使用可以再加入

  4. 最后，单击「Create repository」

建立「Github」的「repository」，完

安装「Hexo」

  1. 安装「node.js」 

    sudo apt-get install nodejs

  2. 安装「npm」 

    sudo apt-get install npm

  3. 安装「Hexo」 

    npm install hexo-cli g

安装「Hexo」，完

安装「Git」

sudo apt-get install git

安装「Git」，完

本地文件上传至「Github」中的「Repository」中

  1. 创建本地「Repository」

    这个怎样说都好，说是本地「Repository」或者是本地仓库，说是「Hexo」目录，都可以。在某个地方，新建个文件夹，比如： 

    mkdir xxx.github.io //目录名与「Github」中的「Repository」同名

  2. 用「Hexo」初始化本地仓库 注1

    cd xxx.github.io //进入本地仓库
hexo init //「Hexo」初始化本地仓库
hexo g //同「hexo generate」生成静态页面和文件
hexo s //开启「Hexo」本地服务器，详见注1

  3. 用「Git」初始化本地仓库

    git init

  4. 安装「Hexo」上传工具 

    npm install hexo-deployer-git --save

  5. 配置「_config.yml」文件 注2

    进入本地仓库后，可以找到「_config.yml」这个文件。内容需要稍作修改。

    其中：

    特别注意：_config.yml中所有冒号后都有空格

    title： //网站标题
subtitle: //网站副标题
...
author: //作者
language: //语言
...
url: //https://usrname.github.io。其实，在仓库设置中可以找到。
root: //就用「/」不要改，如果改了，网站可能没有样式（找不到「CSS」文件夹）

    其中：

    type: git
repo: https://github.com/usrname/usrname.github.io.git //这个在仓库「clone」那里能找到，是个「Https」链接，详见注2
branch: master

  6. 本地仓库中的文件，上传至「Github」中的「Repository」中 

    hexo clean //删除已生成的文件和缓存
hexo g //
hexo d //同「hexo deploy」上传。这个过程中会要过程会要求输入「github」账号和密码

本地文件上传至「Github」中的「Repository」中，完

注1:

如图，浏览器中输入

http://localhost:4000

可以本地浏览网站

若需停止服务，按组合键「Ctrl + C」

执行命令「hexo s」后，报错，错误代码如下或类似：

Port 4000 has been used.//4000端口被占用

处理方法，很简单，重启或注销

注2:

在如图的位置，可以找到仓库的「Https」链接。

以上，全文完。
